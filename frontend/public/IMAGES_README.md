# Placeholder Images for SEO

The following images should be created and placed in `frontend/public/`:

1. **og-image.jpg** - 1200x630px - Open Graph image for social sharing
2. **twitter-image.jpg** - 1200x675px - Twitter Card image  
3. **logo.png** - 600x60px - Site logo
4. **screenshot.jpg** - 1920x1080px - Application screenshot

These images are referenced in:
- `index.html` (OG and Twitter meta tags)
- `SemanticStructure.jsx` (Schema.org structured data)
- `sitemap.xml` (image sitemap)

## Image Requirements

### og-image.jpg
- Size: 1200x630px
- Format: JPG
- Content: Should include site name "Матрица Эйзенхауэра" and visual representation of the matrix
- Purpose: Displayed when sharing links on Facebook, LinkedIn, etc.

### twitter-image.jpg
- Size: 1200x675px
- Format: JPG
- Content: Similar to og-image but optimized for Twitter
- Purpose: Displayed when sharing links on Twitter

### logo.png
- Size: 600x60px (or proportional)
- Format: PNG with transparency
- Content: Site logo/branding
- Purpose: Used in Schema.org Organization markup

### screenshot.jpg
- Size: 1920x1080px
- Format: JPG
- Content: Screenshot of the application interface
- Purpose: Used in SoftwareApplication Schema

## Temporary Solution

Until real images are created, you can use placeholder services or create simple colored images with text using image editing tools or online generators.
